Nivelación Topográfica – Gnomon

Nivelación Topográfica – Gnomon es una aplicación digital diseñada para apoyar los procesos altimétricos utilizados en la topografía moderna. La aplicación permite registrar datos de campo, realizar cálculos automáticos y organizar la información técnica de manera clara y estructurada.

La aplicación integra diferentes herramientas orientadas al trabajo práctico del topógrafo, ingeniero o estudiante, facilitando operaciones comunes de nivelación sin necesidad de realizar cálculos manuales prolongados o utilizar hojas de cálculo externas.

Funciones principales de la aplicación GNOMON.

Entre las principales funciones de la aplicación se encuentran:

  • Nivelación general.
  • Replanteo de niveles.
  • Cálculo de pendientes.
  • Gestión de archivos de trabajo.
  • Exportación y compartición de resultados.

Este manual tiene como objetivo explicar el funcionamiento general de la aplicación y describir el uso de cada uno de sus módulos principales.

Abreviaturas

En los diferentes módulos de la aplicación GNOMON se emplean abreviaturas técnicas propias de la nivelación topográfica. Estos términos permiten interpretar las tablas de cálculo de manera rápida y estandarizada.

Resumen visual de las abreviaturas utilizadas en la aplicación.

Abreviaturas generales

  • PTO: Identificador numérico del punto dentro del proceso de cálculo.
  • V+: Lectura tomada sobre un punto con cota conocida para calcular la altura instrumental.
  • HI: Altura instrumental obtenida al sumar la cota del punto conocido con la vista más (V+).
  • VI: Lecturas realizadas a puntos intermedios desde la misma posición del instrumento.
  • V−: Lectura tomada hacia un punto de cambio o cierre antes de mover el instrumento.
  • Cota: Elevación calculada del punto.
  • Descripción: Identificación del punto (BM, eje, cambio, entre otros).
  • BM: Punto con cota conocida utilizado como referencia inicial.
  • Punto de cambio: Punto utilizado para trasladar la nivelación al cambiar el instrumento.

Replanteo de niveles

  • VI ideal: Lectura necesaria para alcanzar la cota objetivo.
  • Δ (Delta): Diferencia entre la lectura real y la ideal, indica la corrección en campo.
  • Cota Obj: Cota objetivo que se desea replantear.
  • Cota (Lect.): Cota calculada con la lectura real.

Cálculo de pendientes

  • Dist: Distancia entre puntos consecutivos.
  • Pendiente (%): Inclinación definida para el tramo.
  • Cota inicio: Cota base desde la cual inicia el cálculo de la pendiente.

Gestor de archivos

El gestor de archivos es el módulo principal para la organización de proyectos dentro de la aplicación GNOMON. Desde esta sección el usuario puede crear, importar y administrar todos los archivos de trabajo utilizados en los distintos módulos.

Interfaz del gestor de archivos de la aplicación.

Funciones principales

  • Buscar: Permite localizar rápidamente archivos o proyectos mediante el nombre.
  • Filtros: Organización de archivos por categorías como todos, favoritos, locales o recientes.
  • Crear nuevo archivo: Genera un nuevo proyecto para trabajar dentro de cualquier módulo.
  • Subir archivo existente: Permite cargar archivos externos en formato compatible (CSV o TXT).
  • Importar desde otro módulo: Facilita reutilizar información generada en otros procesos dentro de la aplicación.
  • Tarjetas de archivos: Muestran información básica como nombre, fecha y descripción del proyecto.
  • Opciones adicionales: Cada archivo dispone de acciones como editar, eliminar o marcar como favorito.

Este módulo funciona como punto de entrada a todos los procesos de cálculo, permitiendo mantener un flujo de trabajo ordenado y continuo.
